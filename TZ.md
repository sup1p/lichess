1. Описание проекта
Необходимо разработать веб-приложение, позволяющее пользователям
просматривать свою шахматную статистику и историю игр на платформе
Lichess. Приложение должно использовать официальный API Lichess и
обеспечивать удобный интерфейс для анализа игровой активности.

3. Функциональные требования
3.1. Аутентификация через Lichess OAuth2
• Реализовать полный цикл OAuth2 авторизации с Lichess
• Безопасное хранение и обновление токенов доступа
• Кнопка "Войти через Lichess" на главной странице
• Обработка ошибок авторизации с понятными сообщениями
• Возможность выхода из системы (logout)
3.2. Профиль пользователя и рейтинги
После успешной авторизации отобразить:
• Аватар и никнейм пользователя
• Рейтинги по основным типам игр (perfType):
◦ Blitz, Rapid, Classical
• Количество сыгранных партий по каждому типу
• Дата регистрации и последней активности
3.3. История партий
• Таблица/список последних партий с информацией:
◦ Дата и время партии
◦ Тип контроля времени
◦ Соперник (ник и рейтинг)
◦ Результат (победа/поражение/ничья)
◦ Способ завершения (мат, время, сдача)
• Пагинация или бесконечная прокрутка
• Фильтрация по типу игры
• Ссылка для просмотра партии на Lichess
4. Нефункциональные требования
4.1. Интерфейс и адаптивность
• Обязательная адаптивность под мобильные устройства (mobile-first)
4.2. Качество кода
• Типизация: строгая типизация в TypeScript и Pydantic
• Структура проекта: логичное разделение на модули/компоненты
• Обработка ошибок: graceful degradation, информативные сообщения
• Документация: README с инструкциями по запуску
